🎯 VISION DU PROJET
============================================================

CANTAIA est un assistant IA pour les chefs de projet construction en Suisse.
Un logiciel installable (desktop + Outlook) qui automatise le travail administratif,
génère des PV de séance, et offre un tableau de bord multi-chantiers intelligent.

PITCH : "Gagnez 10 heures par semaine. Votre IA trie vos emails par chantier,
rédige vos PV de séance, et vous alerte quand un projet dérape."

MARCHÉ CIBLE : Chefs de projet, conducteurs de travaux, directeurs de travaux
en Suisse romande (puis DACH et France).

LANGUE : Trilingue FR / EN / DE (français par défaut)

MODÈLE ÉCONOMIQUE :
- Phase 1 : 79-149 CHF/mois par utilisateur
- Phase 2 : 199-499 CHF/mois par utilisateur
- Phase 3 : 500-2000 CHF/mois par entreprise

============================================================
🏗️ ARCHITECTURE TECHNIQUE
============================================================

QUATRE SURFACES, UN SEUL BACKEND :

1. APP DESKTOP (Tauri) — .exe installable Windows/Mac
   → Dashboard principal, PV de séance, suivi chantiers, chat (futur)
   → Auto-update intégré Tauri
   → Utilisé quotidiennement au bureau

2. ADD-IN OUTLOOK — Panneau latéral dans Outlook
   → Classement emails par chantier, extraction tâches, briefing quotidien
   → Installé via le store Microsoft Office Add-ins
   → Toujours visible pendant le travail email

3. PWA MOBILE — Web app responsive installable
   → Enregistrement réunions terrain, saisie avancement (Phase 2), chat (Phase 3)
   → Utilisé sur le chantier

4. ADMIN DASHBOARD — Web app accès restreint
   → Vue sur tous les clients, logs d'erreurs, analytics usage, debug
   → Uniquement pour l'équipe interne

| Paiement         | Stripe                     | Abonnements SaaS                           |
| Email            | Resend                     | Emails transactionnels                     |

| Hébergement API  | Vercel                     | Serverless, Edge Functions                 |

CREATE TYPE task_status AS ENUM ('open', 'in_progress', 'waiting', 'completed', 'cancelled');
CREATE TYPE task_priority AS ENUM ('low', 'medium', 'high', 'urgent');
CREATE TYPE task_source AS ENUM ('email', 'meeting_pv', 'manual', 'ai_suggestion');
CREATE TYPE meeting_status AS ENUM ('scheduled', 'recording', 'transcribing', 'generating_pv', 'review', 'finalized', 'sent');
CREATE TYPE email_classification AS ENUM ('action_required', 'info_only', 'urgent', 'waiting_response', 'archived');

------------------------------------------------------------
-- TABLES — CORE (Phase 1+2+3)
------------------------------------------------------------

-- Organisations (entreprises clientes)
CREATE TABLE organizations (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name TEXT NOT NULL,
    address TEXT,
    city TEXT DEFAULT 'Lausanne',
    country TEXT DEFAULT 'CH',
    industry TEXT DEFAULT 'construction',
    subscription_plan subscription_plan DEFAULT 'trial',
    stripe_customer_id TEXT,
    stripe_subscription_id TEXT,
    trial_ends_at TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '14 days'),
    max_users INTEGER DEFAULT 3,
    max_projects INTEGER DEFAULT 5,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Utilisateurs
CREATE TABLE users (
    id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    organization_id UUID REFERENCES organizations(id),
    email TEXT NOT NULL UNIQUE,
    first_name TEXT NOT NULL,
    last_name TEXT NOT NULL,
    role user_role DEFAULT 'project_manager',
    avatar_url TEXT,
    phone TEXT,
    preferred_language TEXT DEFAULT 'fr' CHECK (preferred_language IN ('fr', 'en', 'de')),
    microsoft_access_token TEXT,       -- Token Microsoft Graph API
    microsoft_refresh_token TEXT,
    microsoft_token_expires_at TIMESTAMPTZ,
    outlook_sync_enabled BOOLEAN DEFAULT FALSE,
    last_sync_at TIMESTAMPTZ,
    notification_preferences JSONB DEFAULT '{"email": true, "push": true, "desktop": true}',
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Projets / Chantiers
CREATE TABLE projects (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    organization_id UUID NOT NULL REFERENCES organizations(id),
    created_by UUID NOT NULL REFERENCES users(id),
    name TEXT NOT NULL,                 -- "Campus RTS Ecublens"
    code TEXT,                          -- "RTS-2026" (code interne)
    description TEXT,
    client_name TEXT,                   -- Maître d'ouvrage
    address TEXT,
    city TEXT DEFAULT 'Lausanne',
    status project_status DEFAULT 'active',

    -- Identification emails (mots-clés pour le classement IA)
    email_keywords TEXT[] DEFAULT '{}', -- ['campus rts', 'ecublens', 'fehlmann']
    email_senders TEXT[] DEFAULT '{}',  -- ['architecte@bureau.ch', 'ing@struct.ch']

    -- Dates
    start_date DATE,
    end_date DATE,
    
    -- Phase 2 : Données financières (préparées mais non utilisées en Phase 1)
    budget_total DECIMAL(12,2),
    currency TEXT DEFAULT 'CHF',

    -- Métadonnées
    color TEXT DEFAULT '#6366F1',       -- Couleur du projet dans le dashboard
    archived_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Membres d'un projet (qui a accès)
CREATE TABLE project_members (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role TEXT DEFAULT 'member', -- 'owner', 'member', 'viewer'
    added_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(project_id, user_id)
);

    ai_project_match_confidence DECIMAL(4,2),
    ai_summary TEXT,                            -- Résumé IA en 1-2 phrases

    created_at TIMESTAMPTZ DEFAULT NOW()

📧 INTÉGRATION MICROSOFT OUTLOOK
============================================================

L'intégration Outlook est au cœur du produit. Deux volets :

1. SYNCHRONISATION EMAILS (Backend)

Utilise Microsoft Graph API pour :
- Lire les emails de l'utilisateur (avec son consentement OAuth)
- Écouter les nouveaux emails (webhooks ou polling)
- Classifier et indexer chaque email

Flux :
1. L'utilisateur connecte son compte Microsoft (OAuth 2.0)
2. L'app obtient un access_token + refresh_token
3. Un job récurrent (toutes les 5 min) récupère les nouveaux emails
4. Chaque email est envoyé à l'IA pour classement
5. Les tâches détectées sont créées automatiquement

Configuration Microsoft :
- Enregistrer une app sur Azure AD (portal.azure.com)
- Permissions requises : Mail.Read, Mail.ReadBasic, Calendars.Read, User.Read
- Redirect URI pour l'OAuth

Code clé :
```typescript
// Microsoft Graph - Récupérer les derniers emails
const response = await graphClient
  .api('/me/messages')
  .select('id,subject,from,toRecipients,receivedDateTime,bodyPreview,hasAttachments')
  .filter(`receivedDateTime ge ${lastSyncDate}`)
  .orderby('receivedDateTime desc')
  .top(50)
  .get();
```

2. ADD-IN OUTLOOK (Frontend)

Panneau latéral qui affiche :
- Le projet détecté pour l'email ouvert
- Les tâches extraites de cet email
- Le briefing du jour
- Les actions rapides (reclasser, créer tâche, marquer urgent)
